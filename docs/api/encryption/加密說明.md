# 加密相關說明

## 概述

API 的請求與回應內容需經過 AES 256 加密處理，並使用 `RequestData` 和 `ReturnData` 結構包裹加密後的數據。

## 加密規格

| 項目 | 說明 |
|------|------|
| **加密演算法** | AES 256 |
| **加密模式** | CBC (Cipher Block Chaining) |
| **填充方式** | PKCS7 |
| **金鑰 (AES Key)** | 後續提供 (至少 22 bytes) |
| **初始化向量 (IV)** | 從 AES Key 的第 7~22 字元提取 (16 字元) |
| **金鑰長度** | 256 bits (32 bytes) |
| **區塊大小** | 128 bits (16 bytes) |

**IV 生成說明**：
- IV 不需要單獨提供
- 直接從 AES Key 的第 7~22 字元（即索引 6~21，共 16 個字元）提取作為 IV
- 例如：AES Key = `"Your32ByteKeyHere1234567890123456"`
  - AES Key (前 32 bytes): `"Your32ByteKeyHere12345678901"`
  - IV (第 7~22 字元): `"yKeyHere123456789"`

## 請求與回應結構

### 請求結構

**外層格式**：
```json
{
  "RequestData": "AES256加密後的數據"
}
```

**加密流程**：
1. 準備內層 JSON 數據（例如登入請求）
2. 將內層 JSON 字串進行 minimize（移除多餘空白）
3. 將 minimize 後的字串進行 AES 256 加密
   - 使用 AES Key 的前 32 字元作為加密金鑰
   - 使用 AES Key 的第 7~22 字元作為初始化向量 (IV)
4. 將加密後的二進制數據轉換為 Base64 字串
5. 將 Base64 字串放置於外層請求的 `RequestData` 欄位中

### 回應結構

**外層格式**：
```json
{
  "ReturnData": "AES256加密後的數據"
}
```

**解密流程**：
1. 從外層回應的 `ReturnData` 欄位中提取 Base64 字串
2. 將 Base64 字串轉換回二進制數據
3. 對二進制數據進行 AES 256 解密
   - 使用 AES Key 的前 32 字元作為加密金鑰
   - 使用 AES Key 的第 7~22 字元作為初始化向量 (IV)
4. 解密後的字串即為內層 JSON 回應數據

## 實作範例

### C# AES 加解密

```csharp
using System;
using System.IO;
using System.Security.Cryptography;
using System.Text;

public static class AesHelper
{
    /// <summary>
    /// 從 AES Key 中提取 IV (第 7~22 字元)
    /// </summary>
    private static byte[] ExtractIvFromKey(byte[] key)
    {
        // IV 從 key 的索引 6~21 (第 7~22 字元) 提取
        byte[] iv = new byte[16];
        int keyLength = Math.Min(key.Length, 22);
        int start = 6;
        for (int i = 0; i < 16 && (start + i) < keyLength; i++)
        {
            iv[i] = key[start + i];
        }
        // 如果 key 長度不足 22，用 0 補足
        for (int i = keyLength - start; i < 16; i++)
            iv[i] = 0;
        return iv;
    }

    /// <summary>
    /// 確保 AES Key 至少有 32 bytes (256 bits)
    /// </summary>
    private static byte[] EnsureKeyLength(byte[] key)
    {
        byte[] fullKey = new byte[32];
        int copyLength = Math.Min(key.Length, 32);
        Array.Copy(key, 0, fullKey, 0, copyLength);
        // 如果 key 長度不足 32，用 0 補足
        return fullKey;
    }

    /// <summary>
    /// 將 JSON 物件加密並塞入 RequestData 結構
    /// </summary>
    public static string EncryptRequest(object jsonData, byte[] key)
    {
        // 1. 將物件序列化為 JSON 字串
        string jsonString = JsonSerializer.Serialize(jsonData, new JsonSerializerOptions
        {
            WriteIndented = false  // minimize JSON
        });

        // 2. 準備 AES Key (32 bytes) 和從中提取 IV (16 bytes)
        byte[] fullKey = EnsureKeyLength(key);
        byte[] iv = ExtractIvFromKey(key);

        // 3. AES 256 加密
        string encryptedString = Encrypt(jsonString, fullKey, iv);

        // 4. 構造外層請求
        var outerRequest = new { RequestData = encryptedString };
        return JsonSerializer.Serialize(outerRequest, new JsonSerializerOptions
        {
            WriteIndented = false
        });
    }

    /// <summary>
    /// 從 ReturnData 結構中解密回應
    /// </summary>
    public static T DecryptResponse<T>(string responseJson, byte[] key)
    {
        // 1. 解析外層回應
        var outerResponse = JsonSerializer.Deserialize<ReturnDataWrapper>(responseJson);
        if (outerResponse?.ReturnData == null)
            throw new Exception("Invalid response format");

        // 2. 準備 AES Key (32 bytes) 和從中提取 IV (16 bytes)
        byte[] fullKey = EnsureKeyLength(key);
        byte[] iv = ExtractIvFromKey(key);

        // 3. 解密 ReturnData
        string decryptedString = Decrypt(outerResponse.ReturnData, fullKey, iv);

        // 4. 反序列化為回應物件
        return JsonSerializer.Deserialize<T>(decryptedString) 
               ?? throw new Exception("Failed to deserialize response");
    }

    public static string Encrypt(string plainText, byte[] key, byte[] iv)
    {
        using var aes = Aes.Create();
        aes.Key = key;
        aes.IV = iv;
        aes.Mode = CipherMode.CBC;
        aes.Padding = PaddingMode.PKCS7;

        using var encryptor = aes.CreateEncryptor();
        using var ms = new MemoryStream();
        using (var cs = new CryptoStream(ms, encryptor, CryptoStreamMode.Write))
        using (var sw = new StreamWriter(cs))
        {
            sw.Write(plainText);
        }
        return Convert.ToBase64String(ms.ToArray());
    }

    public static string Decrypt(string cipherText, byte[] key, byte[] iv)
    {
        using var aes = Aes.Create();
        aes.Key = key;
        aes.IV = iv;
        aes.Mode = CipherMode.CBC;
        aes.Padding = PaddingMode.PKCS7;

        using var decryptor = aes.CreateDecryptor();
        using var ms = new MemoryStream(Convert.FromBase64String(cipherText));
        using var cs = new CryptoStream(ms, decryptor, CryptoStreamMode.Read);
        using var sr = new StreamReader(cs);
        return sr.ReadToEnd();
    }

    // 字串轉 byte[]
    public static byte[] StringToBytes(string str)
    {
        return Encoding.UTF8.GetBytes(str);
    }

    public class ReturnDataWrapper
    {
        public string ReturnData { get; set; } = string.Empty;
    }
}
```

#### C# 使用範例

```csharp
// 假設從配置文件或安全儲存中獲取 AES Key
// AES Key 至少需要 22 個字元 (bytes)，前 32 個字元作為金鑰，第 7~22 個字元作為 IV
string aesKeyString = "Your32ByteKeyHere1234567890123456"; // 至少 22 個字元

byte[] aesKey = AesHelper.StringToBytes(aesKeyString);

// ===== 登入請求範例 =====
var loginData = new {
    requestId = Guid.NewGuid().ToString(),
    account = "user001",
    password = "password123"
};

// 加密並構造外層請求 (IV 會自動從 AES Key 的第 7~22 字元提取)
string encryptedRequest = AesHelper.EncryptRequest(loginData, aesKey);
Console.WriteLine("請求: " + encryptedRequest);
// 輸出: {"RequestData":"8uhB22wljs/Zqm4jfDrcVKIHQc0cBwjkgr..."}

// ===== 發送請求至 API =====
// ... 使用 HttpClient 發送 encryptedRequest ...

// ===== 解密回應範例 =====
string apiResponse = "{\"ReturnData\":\"8uhB22wljs/Zqm4jfDrcVKIHQc0cBwjkgr...\"}";

// 解密回應 (IV 會自動從 AES Key 的第 7~22 字元提取)
var responseData = AesHelper.DeserializeResponse<LoginResponse>(apiResponse, aesKey);
Console.WriteLine("狀態: " + responseData.status);
Console.WriteLine("訊息: " + responseData.message);
Console.WriteLine("Token: " + responseData.data.accessToken);
```

### Android (Java/Kotlin) AES 加解密

#### Java 版本

```java
import javax.crypto.*;
import javax.crypto.spec.IvParameterSpec;
import javax.crypto.spec.SecretKeySpec;
import java.util.Base64;
import java.nio.charset.StandardCharsets;

public class AesUtil {
    private static final String ALGORITHM = "AES/CBC/PKCS7Padding";

    /// <summary>
    /// 從 AES Key 中提取 IV (第 7~22 字元)
    /// </summary>
    private static byte[] extractIvFromKey(byte[] key) {
        // IV 從 key 的索引 6~21 (第 7~22 字元) 提取
        byte[] iv = new byte[16];
        int keyLength = Math.min(key.length, 22);
        int start = 6;
        for (int i = 0; i < 16 && (start + i) < keyLength; i++) {
            iv[i] = key[start + i];
        }
        // 如果 key 長度不足 22，用 0 補足
        for (int i = keyLength - start; i < 16; i++) {
            iv[i] = 0;
        }
        return iv;
    }

    /// <summary>
    /// 確保 AES Key 至少有 32 bytes (256 bits)
    /// </summary>
    private static byte[] ensureKeyLength(byte[] key) {
        byte[] fullKey = new byte[32];
        int copyLength = Math.min(key.length, 32);
        System.arraycopy(key, 0, fullKey, 0, copyLength);
        // 如果 key 長度不足 32，用 0 補足
        return fullKey;
    }

    /// <summary>
    /// 加密並構造 RequestData 結構
    /// </summary>
    public static String encryptRequest(Object jsonData, byte[] key) throws Exception {
        // 1. 將物件序列化為 JSON 字串 (minimize)
        String jsonString = new Gson().toJson(jsonData); // 使用 minimize 設定

        // 2. 準備 AES Key (32 bytes) 和從中提取 IV (16 bytes)
        byte[] fullKey = ensureKeyLength(key);
        byte[] iv = extractIvFromKey(key);

        // 3. AES 256 加密
        String encryptedString = encrypt(jsonString, fullKey, iv);

        // 4. 構造外層請求
        Map<String, String> outerRequest = new HashMap<>();
        outerRequest.put("RequestData", encryptedString);
        return new Gson().toJson(outerRequest);
    }

    /// <summary>
    /// 從 ReturnData 結構中解密回應
    /// </summary>
    public static <T> T deserializeResponse(String responseJson, Class<T> responseClass, byte[] key) throws Exception {
        // 1. 解析外層回應
        Type type = new TypeToken<Map<String, String>>(){}.getType();
        Map<String, String> outerResponse = new Gson().fromJson(responseJson, type);

        if (outerResponse == null || !outerResponse.containsKey("ReturnData")) {
            throw new Exception("Invalid response format");
        }

        // 2. 準備 AES Key (32 bytes) 和從中提取 IV (16 bytes)
        byte[] fullKey = ensureKeyLength(key);
        byte[] iv = extractIvFromKey(key);

        // 3. 解密 ReturnData
        String decryptedString = decrypt(outerResponse.get("ReturnData"), fullKey, iv);

        // 4. 反序列化為回應物件
        return new Gson().fromJson(decryptedString, responseClass);
    }

    public static String encrypt(String plainText, byte[] key, byte[] iv) throws Exception {
        Cipher cipher = Cipher.getInstance(ALGORITHM);
        SecretKeySpec keySpec = new SecretKeySpec(key, "AES");
        IvParameterSpec ivSpec = new IvParameterSpec(iv);
        cipher.init(Cipher.ENCRYPT_MODE, keySpec, ivSpec);
        byte[] encrypted = cipher.doFinal(plainText.getBytes(StandardCharsets.UTF_8));
        return Base64.encodeToString(encrypted, Base64.NO_WRAP);
    }

    public static String decrypt(String cipherText, byte[] key, byte[] iv) throws Exception {
        Cipher cipher = Cipher.getInstance(ALGORITHM);
        SecretKeySpec keySpec = new SecretKeySpec(key, "AES");
        IvParameterSpec ivSpec = new IvParameterSpec(iv);
        cipher.init(Cipher.DECRYPT_MODE, keySpec, ivSpec);
        byte[] decoded = Base64.decode(cipherText, Base64.NO_WRAP);
        byte[] decrypted = cipher.doFinal(decoded);
        return new String(decrypted, StandardCharsets.UTF_8);
    }

    // 字串轉 byte[]
    public static byte[] stringToBytes(String str) {
        return str.getBytes(StandardCharsets.UTF_8);
    }
}
```

#### Kotlin 版本

```kotlin
import javax.crypto.Cipher
import javax.crypto.spec.IvParameterSpec
import javax.crypto.spec.SecretKeySpec
import java.util.Base64
import java.nio.charset.StandardCharsets
import com.google.gson.Gson
import com.google.gson.GsonBuilder
import com.google.gson.reflect.TypeToken

object AesUtil {
    private const val ALGORITHM = "AES/CBC/PKCS7Padding"
    private val gson: Gson = GsonBuilder().create()

    /// <summary>
    /// 從 AES Key 中提取 IV (第 7~22 字元)
    /// </summary>
    private fun extractIvFromKey(key: ByteArray): ByteArray {
        // IV 從 key 的索引 6~21 (第 7~22 字元) 提取
        val iv = ByteArray(16)
        val keyLength = minOf(key.size, 22)
        val start = 6
        for (i in 0 until 16 && (start + i) < keyLength) {
            iv[i] = key[start + i]
        }
        // 如果 key 長度不足 22，用 0 補足
        for (i in (keyLength - start) until 16) {
            iv[i] = 0
        }
        return iv
    }

    /// <summary>
    /// 確保 AES Key 至少有 32 bytes (256 bits)
    /// </summary>
    private fun ensureKeyLength(key: ByteArray): ByteArray {
        val fullKey = ByteArray(32)
        val copyLength = minOf(key.size, 32)
        System.arraycopy(key, 0, fullKey, 0, copyLength)
        // 如果 key 長度不足 32，用 0 補足
        return fullKey
    }

    /// <summary>
    /// 加密並構造 RequestData 結構
    /// </summary>
    fun encryptRequest(jsonData: Any, key: ByteArray): String {
        // 1. 將物件序列化為 JSON 字串 (minimize)
        val jsonString = gson.toJson(jsonData)

        // 2. 準備 AES Key (32 bytes) 和從中提取 IV (16 bytes)
        val fullKey = ensureKeyLength(key)
        val iv = extractIvFromKey(key)

        // 3. AES 256 加密
        val encryptedString = encrypt(jsonString, fullKey, iv)

        // 4. 構造外層請求
        val outerRequest = mapOf("RequestData" to encryptedString)
        return gson.toJson(outerRequest)
    }

    /// <summary>
    /// 從 ReturnData 結構中解密回應
    /// </summary>
    inline fun <reified T> deserializeResponse(responseJson: String, key: ByteArray): T {
        // 1. 解析外層回應
        val type = object : TypeToken<Map<String, String>>() {}.type
        val outerResponse: Map<String, String> = gson.fromJson(responseJson, type)

        if (!outerResponse.containsKey("ReturnData")) {
            throw Exception("Invalid response format")
        }

        // 2. 準備 AES Key (32 bytes) 和從中提取 IV (16 bytes)
        val fullKey = ensureKeyLength(key)
        val iv = extractIvFromKey(key)

        // 3. 解密 ReturnData
        val decryptedString = decrypt(outerResponse["ReturnData"]!!, fullKey, iv)

        // 4. 反序列化為回應物件
        return gson.fromJson(decryptedString, T::class.java)
    }

    fun encrypt(plainText: String, key: ByteArray, iv: ByteArray): String {
        val cipher = Cipher.getInstance(ALGORITHM)
        val keySpec = SecretKeySpec(key, "AES")
        val ivSpec = IvParameterSpec(iv)
        cipher.init(Cipher.ENCRYPT_MODE, keySpec, ivSpec)
        val encrypted = cipher.doFinal(plainText.toByteArray(StandardCharsets.UTF_8))
        return Base64.encodeToString(encrypted, Base64.NO_WRAP)
    }

    fun decrypt(cipherText: String, key: ByteArray, iv: ByteArray): String {
        val cipher = Cipher.getInstance(ALGORITHM)
        val keySpec = SecretKeySpec(key, "AES")
        val ivSpec = IvParameterSpec(iv)
        cipher.init(Cipher.DECRYPT_MODE, keySpec, ivSpec)
        val decoded = Base64.decode(cipherText, Base64.NO_WRAP)
        val decrypted = cipher.doFinal(decoded)
        return String(decrypted, StandardCharsets.UTF_8)
    }

    // 字串轉 byte[]
    fun stringToBytes(str: String): ByteArray {
        return str.toByteArray(StandardCharsets.UTF_8)
    }
}
```

## 注意事項

1. **金鑰管理**
   - AES Key 應妥善保管，不應硬編碼在程式碼中
   - 建議使用安全的儲存機制(如 Android 的 Keystore、.NET 的 DPAPI)
   - 金鑰應定期更新
   - IV 不需要單獨管理，會從 AES Key 中自動提取

2. **加密時機**
   - 請求加密:將整個 JSON 請求體加密後放置於 `RequestData` 欄位中
   - 回應解密: 從 `ReturnData` 欄位提取並解密內容
   - AES Key 的前 32 字元作為加密金鑰，第 7~22 字元作為初始化向量

3. **JSON Minimize**
   - 加密前的 JSON 應進行 minimize（移除多餘空白）處理
   - 可使用 `JsonSerializerOptions.WriteIndented = false` (C#) 或 Gson 預設最小化 (Java/Kotlin)

4. **錯誤處理**
   - 解密失敗可能表示:
     - AES Key 錯誤
     - 資料被篡改
     - 格式錯誤
   - 應適當處理加密/解密異常

5. **安全性**
   - 使用 HTTPS 傳輸加密後的資料
   - AES Key 長度至少需要 22 個字元 (bytes)，建議使用 32 個字元以上
   - IV 將從 AES Key 中提取，不可自行更改

6. **統一結構**
   - 所有 API 接口（除登入接口外）必須使用 `RequestData`/`ReturnData` 結構
   - 登入接口同樣使用此結構進行加解密

## 未提供資訊

目前加密流程中的具體數值尚未提供:
- **AES Key**: 待系統提供 (至少 22 bytes 的字串，建議 32 bytes 以上)

實作時請使用動態配置或從安全儲存中讀取 AES Key。IV 會自動從 AES Key 的第 7~22 字元提取。
