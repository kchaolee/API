R3 - 回傳收貨資料

接口資訊

項目 內容
接口名稱 回傳收貨資料
接口代號 R3
URL POST /wmService/v1/PO/PoReceivingItem
通訊方式 POST
請求格式 JSON
回應格式 JSON
提供方 WMS
調用方 外部系統 (Android App)

功能說明

輸入收貨資料並暫存至 WMS。同一 lineNo 可多次更新。透過 requestFnName 區分「3.2確認收貨」(新增) 與「3.3修改」(更新) 操作。

請求參數

項次 參數名稱 參數說明 傳輸型態 DB資料類型 字串最大長度 備註 必填
1 requestId 請求編號 String(36) GUID 36 每個請求唯一，重複提交使用同一編號 V
2 requestFnName 請求來源 String(30) VarChar 30 依畫面功能填入：“3.2確認收貨”、“3.3修改” V
3 wmsAsnNumber WMS預期收貨單號 String(10) Char 10 帶 R1 回傳的 “WMS預期收貨單號” V
4 storerCode 貨主代碼 String(15) Char 15 帶 R1 回傳的 “貨主代碼” V
5 externReceiptNumber 外部收貨編號 String(30) Char 30 外部收貨編號＝採購單號 V
6 lineNo WMS項次 String(5) Char 5 WMS項次，明細主鍵值 V
7 externLineNo 外部項次 String(20) Char 20 外部項次＝PO項次 V
8 sku 貨號 String(50) Char 50 - V
9 expiryDate 到期日 String(10) DateTime - 格式：“yyyy/MM/dd” V
10 packQty 件數 String(10) int - 件數，需為整數 V
11 qty 數量 String(20) decimal(19,5) - WMS數量，小數 條件式必填
12 batchNumber 生產批號 String(50) Char 50 - 條件式必填
13 mfgDate 製造日 String(10) DateTime - 格式：“yyyy/MM/dd” 條件式必填
14 storageStatus 倉別 String(50) Char 50 - 條件式必填
15 stockType 區位 String(50) Char 50 - 條件式必填
16 other 其他 String(50) Char 50 - 條件式必填
17 other1 其他1 String(50) Char 50 - 條件式必填

條件式必填欄位說明

以下欄位是否必填，取決於 R2 回傳的 itemColumnConfig[colRequired]：

  • qty: 依 itemColumnConfig 中 colName="qty" 的 colRequired 值決定
  • batchNumber: 依 itemColumnConfig 中 colName="batchNumber" 的 colRequired 值決定
  • mfgDate: 依 itemColumnConfig 中 colName="mfgDate" 的 colRequired 值決定
  • storageStatus: 依 itemColumnConfig 中 colName="storageStatus" 的 colRequired 值決定
  • stockType: 依 itemColumnConfig 中 colName="stockType" 的 colRequired 值決定

Android Kotlin 範例

data class ReceivingInput(
    val lineNo: String,
    val externLineNo: String,
    val sku: String,
    val expiryDate: String,
    val packQty: Int,
    val qty: Double? = null,
    val batchNumber: String? = null,
    val mfgDate: String? = null,
    val storageStatus: String? = null,
    val stockType: String? = null,
    val other: String? = null,
    val other1: String? = null
)

class ReceivingRepository {
    private val baseUrl = "http://192.168.x.x/wmService/v1"
    private val client = OkHttpClient()

    suspend fun saveReceivingItem(
        wmsAsnNumber: String,
        storerCode: String,
        externReceiptNumber: String,
        input: ReceivingInput,
        requestFnName: RequestFnName
    ): Result<Unit> = withContext(Dispatchers.IO) {
        val requestId = UUID.randomUUID().toString()
        val token = getAccessToken()

        try {
            val jsonRequest = JSONObject().apply {
                put("requestId", requestId)
                put("requestFnName", requestFnName.value)
                put("wmsAsnNumber", wmsAsnNumber)
                put("storerCode", storerCode)
                put("externReceiptNumber", externReceiptNumber)
                put("lineNo", input.lineNo)
                put("externLineNo", input.externLineNo)
                put("sku", input.sku)
                put("expiryDate", input.expiryDate)
                put("packQty", input.packQty.toString())

                // 條件式必填欄位
                input.qty?.let { put("qty", it.toString()) }
                input.batchNumber?.let { put("batchNumber", it) }
                input.mfgDate?.let { put("mfgDate", it) }
                input.storageStatus?.let { put("storageStatus", it) }
                input.stockType?.let { put("stockType", it) }
                input.other?.let { put("other", it) }
                input.other1?.let { put("other1", it) }
            }

            val body = jsonRequest.toString().toRequestBody("application/json".toMediaType())
            val request = Request.Builder()
                .url("$baseUrl/PO/PoReceivingItem")
                .addHeader("Authorization", "Bearer $token")
                .post(body)
                .build()

            val response = client.newCall(request).execute()
            val responseBody = response.body?.string() ?: throw Exception("無回應")

            val jsonResponse = JSONObject(responseBody)

            if (jsonResponse.getString("status") == "S") {
                Result.success(Unit)
            } else {
                val message = jsonResponse.getString("message")
                Result.failure(Exception(message))
            }
        } catch (e: Exception) {
            Result.failure(e)
        }
    }

    // 確認收貨 (新增)
    suspend fun confirmReceiving(
        wmsAsnNumber: String,
        storerCode: String,
        externReceiptNumber: String,
        input: ReceivingInput
    ) = saveReceivingItem(wmsAsnNumber, storerCode, externReceiptNumber, input, RequestFnName.CONFIRM)

    // 修改收貨 (更新)
    suspend fun modifyReceiving(
        wmsAsnNumber: String,
        storerCode: String,
        externReceiptNumber: String,
        input: ReceivingInput
    ) = saveReceivingItem(wmsAsnNumber, storerCode, externReceiptNumber, input, RequestFnName.MODIFY)
}

enum class RequestFnName(val value: String) {
    CONFIRM("3.2確認收貨"),
    MODIFY("3.3修改")
}

關聯接口

此接口的輸入來自 R2，輸出為 R4 的前置條件：

上一個接口 下一個接口
R2 - 取得收貨明細資料 R4 - 取得收貨核對明細

RequestFnName 區分說明

requestFnName 說明 行為
3.2確認收貨 首次輸入該明細的收貨資料 新增收貨暫存資料
3.3修改 修改已輸入的收貨資料 更新收貨暫存資料

錯誤代碼

錯誤代碼 HTTP 說明 因應方式
F119 401 Token 驗證失效 重新登入取得 Token
F981 400 資料格式錯誤 檢查輸入資料
F983 400 採購單已過帳 該採購單已完成，無法修改
F999 500 伺服器異常 聯繫技術支援

注意事項

  1. RequestFnName 判斷:

    • 首次輸入某明細的收貨資料，使用 requestFnName = "3.2確認收貨"
    • 修改已輸入的收貨資料，使用 requestFnName = "3.3修改"
    • 同一 lineNo 第二次傳送視為更新

  2. 件數格式:

    • packQty 需為整數，傳遞時建議使用字串格式
    • 如："100" 而非 100.5

  3. 條件式必填欄位:

    • R2 回傳的 itemColumnConfig 決定哪些欄位為必填
    • 必填欄位在 R3 呼叫時必須傳送，否則會收到 F981 錯誤

  4. 資料暫存:

    • R3 的資料是暫存，未過帳前可修改多次
    • R5 全收確認後才正式過帳

  5. 重複提交防護:

    • 同一操作的重試請求應使用相同的 requestId
    • 避免重複提交造成資料不一致

  6. 狀態變更:

    • 首次 R3 確認收貨後，採購單狀態變更為「收貨中」
    • 狀態變更後，R5 才能被調用
