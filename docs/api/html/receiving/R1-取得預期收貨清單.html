R1 - 取得預期收貨清單

接口資訊

項目 內容
接口名稱 取得預期收貨清單
接口代號 R1
URL POST /wmService/v1/PO/PoHeaderData
通訊方式 POST
請求格式 JSON
回應格式 JSON
提供方 WMS
調用方 外部系統 (Android App)

功能說明

取得未完成收貨的採購單清單，用戶從清單中選擇要收貨的採購單。僅回傳狀態為「OPEN」的採購單。

請求參數

項次 參數名稱 參數說明 傳輸型態 DB資料類型 字串最大長度 備註 必填
1 requestId 請求編號 String(36) GUID 36 每個請求唯一，重複提交使用同一編號 V
2 storerCode 貨主代碼 String(15) Char 15 固定帶 “97286918” V
3 docStatus 單據狀態 String(10) Char 10 固定帶 “OPEN” V

請求範例

{
  "requestId": "fb1a6bb8-21cd-490e-9f47-962cf99ec089",
  "storerCode": "97286918",
  "docStatus": "OPEN"
}

JSON Schema

interface R1Request {
  requestId: string;        // 格式: UUID v4
  storerCode: "97286918";   // 固定值
  docStatus: "OPEN";        // 固定值
}

回應參數

項次 參數名稱 參數說明 傳輸型態 DB資料類型 字串最大長度 備註 必填
1 requestId 請求編號 String(36) Char 36 傳回原始請求編號 V
2 status API回應結果代碼 String(1) Char - 成功代碼：S、失敗代碼：F V
3 message API回應結果訊息 String(255) Char 255 - V
4 data 回覆資料內容 Object - - 成功時有值 -
5 headerList 表頭清單資料列 Array - - 採購單清單陣列 -
6 warehouseCode 倉庫 String(10) Char 10 實體倉庫代碼＝回廠工廠 V
7 wmsAsnNumber WMS預期收貨單號 String(10) Char 10 WMS單號主鍵值，後續API傳送需使用 V
8 storerCode 貨主代碼 String(15) Char 15 貨主主檔代碼 V
9 externReceiptNumber 外部收貨編號 String(30) Char 30 外部收貨編號＝採購單號 V
10 vendorName 供應商名稱 String(30) Char 30 - V
11 errors 錯誤訊息內容 Array - - 失敗時有值 -

使用方式

Curl 範例

curl -X POST http://192.168.x.x/wmService/v1/PO/PoHeaderData \
  -H "Authorization: Bearer {accessToken}" \
  -H "Content-Type: application/json" \
  -d '{
    "requestId": "fb1a6bb8-21cd-490e-9f47-962cf99ec089",
    "storerCode": "97286918",
    "docStatus": "OPEN"
  }'

Android Java 範例

public class ReceivingRepository {
    private static final String BASE_URL = "http://192.168.x.x/wmService/v1";
    private static final MediaType JSON = MediaType.get("application/json; charset=utf-8");

    public List<ReceivingHeader> getReceivingHeaders() throws Exception {
        String requestId = UUID.randomUUID().toString();
        String token = getAccessToken(); // 從儲存中取得 Token

        JSONObject jsonRequest = new JSONObject();
        jsonRequest.put("requestId", requestId);
        jsonRequest.put("storerCode", "97286918");
        jsonRequest.put("docStatus", "OPEN");

        RequestBody body = RequestBody.create(jsonRequest.toString(), JSON);
        Request request = new Request.Builder()
            .url(BASE_URL + "/PO/PoHeaderData")
            .addHeader("Authorization", "Bearer " + token)
            .post(body)
            .build();

        OkHttpClient client = new OkHttpClient();
        Response response = client.newCall(request).execute();
        String responseBody = response.body().string();

        JSONObject jsonResponse = new JSONObject(responseBody);

        if ("S".equals(jsonResponse.getString("status"))) {
            JSONArray headerList = jsonResponse.getJSONObject("data").getJSONArray("headerList");
            List<ReceivingHeader> headers = new ArrayList<>();

            for (int i = 0; i < headerList.length(); i++) {
                JSONObject header = headerList.getJSONObject(i);
                headers.add(new ReceivingHeader(
                    header.getString("warehouseCode"),
                    header.getString("wmsAsnNumber"),
                    header.getString("storerCode"),
                    header.getString("externReceiptNumber"),
                    header.getString("vendorName")
                ));
            }

            return headers;
        } else {
            throw new Exception(jsonResponse.getString("message"));
        }
    }

    public static class ReceivingHeader {
        public String warehouseCode;
        public String wmsAsnNumber;
        public String storerCode;
        public String externReceiptNumber;
        public String vendorName;

        // 建構函式、getter、setter...
    }
}

Android Kotlin 範例

data class ReceivingHeader(
    val warehouseCode: String,
    val wmsAsnNumber: String,
    val storerCode: String,
    val externReceiptNumber: String,
    val vendorName: String
)

class ReceivingRepository {
    private val baseUrl = "http://192.168.x.x/wmService/v1"
    private val client = OkHttpClient()

    suspend fun getReceivingHeaders(): Result<List<ReceivingHeader>> = withContext(Dispatchers.IO) {
        val requestId = UUID.randomUUID().toString()
        val token = getAccessToken()

        try {
            val jsonRequest = JSONObject().apply {
                put("requestId", requestId)
                put("storerCode", "97286918")
                put("docStatus", "OPEN")
            }

            val body = jsonRequest.toString().toRequestBody("application/json".toMediaType())
            val request = Request.Builder()
                .url("$baseUrl/PO/PoHeaderData")
                .addHeader("Authorization", "Bearer $token")
                .post(body)
                .build()

            val response = client.newCall(request).execute()
            val responseBody = response.body?.string() ?: throw Exception("無回應")

            val jsonResponse = JSONObject(responseBody)

            if (jsonResponse.getString("status") == "S") {
                val headerList = jsonResponse.getJSONObject("data")
                    .getJSONArray("headerList")

                val headers = mutableListOf<ReceivingHeader>()
                for (i in 0 until headerList.length()) {
                    val header = headerList.getJSONObject(i)
                    headers.add(
                        ReceivingHeader(
                            warehouseCode = header.getString("warehouseCode"),
                            wmsAsnNumber = header.getString("wmsAsnNumber"),
                            storerCode = header.getString("storerCode"),
                            externReceiptNumber = header.getString("externReceiptNumber"),
                            vendorName = header.getString("vendorName")
                        )
                    )
                }

                Result.success(headers)
            } else {
                val message = jsonResponse.getString("message")
                Result.failure(Exception(message))
            }
        } catch (e: Exception) {
            Result.failure(e)
        }
    }
}

關聯接口

此接口為進貨作業的起始點，成功後需調用以下接口：

下一個接口 說明
R2 - 取得收貨明細資料 使用 wmsAsnNumber、storerCode、externReceiptNumber 取得採購單明細

錯誤代碼

錯誤代碼 HTTP 說明 因應方式
F119 401 Token 驗證失效 重新登入取得 Token
F982 404 單據狀態不存在 檢查 docStatus 參數值
F999 500 伺服器異常 聯繫技術支援

注意事項

  1. Token 傳遞: 必須在 HTTP Header 中加入 Authorization: Bearer {accessToken}

  2. 回傳欄位儲存:

    • wmsAsnNumber: 關鍵欄位，後續 R2-R5 都需要使用
    • storerCode: 後續 API 唯一標識
    • externReceiptNumber: 採購單號，應用層展示使用

  3. 無資料處理:

    • 當 headerList 為空陣列時，表示當前無待收貨的採購單
    • 應顯示「目前無待收貨採購單」訊息給用戶

  4. 請求重試:

    • 同一操作的重試請求應使用相同的 requestId
    • 避免重複調用造成伺服端混淆

  5. 排序建議:

    • 建議用戶端依 externReceiptNumber 或 vendorName 排序展示
    • 方便用戶快速找到要處理的採購單
