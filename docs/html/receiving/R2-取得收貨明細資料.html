R2 - 取得收貨明細資料

接口資訊

項目 內容
接口名稱 取得收貨明細資料
接口代號 R2
URL POST /wmService/v1/PO/PoDetailData
通訊方式 POST
請求格式 JSON
回應格式 JSON
提供方 WMS
調用方 外部系統 (Android App)

功能說明

依採購單號取得收貨明細，包含 PO 項次、貨號、品名、到期日、漁廠代碼等資訊。使用者根據這些明細逐項輸入收貨資料。

請求參數

項次 參數名稱 參數說明 傳輸型態 DB資料類型 字串最大長度 備註 必填
1 requestId 請求編號 String(36) GUID 36 每個請求唯一，重複提交使用同一編號 V
2 wmsAsnNumber WMS預期收貨單號 String(10) Char 10 帶 R1 回傳的 “WMS預期收貨單號” V
3 storerCode 貨主代碼 String(15) Char 15 帶 R1 回傳的 “貨主代碼” V
4 externReceiptNumber 外部收貨編號 String(30) Char 30 外部收貨編號＝採購單號 V

請求範例

{
  "requestId": "fb1a6bb8-21cd-490e-9f47-962cf99ec089",
  "wmsAsnNumber": "0000000001",
  "storerCode": "69512619",
  "externReceiptNumber": "API21052600601"
}

interface R2Request {
  requestId: string;        // 格式: UUID v4
  wmsAsnNumber: string;    // 來自 R1 回傳
  storerCode: string;      // 來自 R1 回傳
  externReceiptNumber: string; // 來自 R1 回傳

回應參數

項次 參數名稱 參數說明 傳輸型態 DB資料類型 字串最大長度 備註 必填
1 requestId 請求編號 String(36) Char 36 傳回原始請求編號 V
2 status API回應結果代碼 String(1) Char - 成功代碼：S、失敗代碼：F V
3 message API回應結果訊息 String(255) Char 255 - V
4 data 回覆資料內容 Object - - 成功時有值 -
5 detailList 明細資料陣列 Array - - 收貨明細項目清單 -
6 lineNo WMS項次 String(5) Char 5 WMS 項次，主鍵值 V
7 externLineNo 外部項次 String(20) Char 20 外部項次＝PO項次 V
8 sku 貨號 String(50) Char 50 - V
9 itemName 品名 String(200) NVarChar 200 - V
10 expiryDate 到期日 String(10) DateTime - 格式：“yyyy/MM/dd” -
11 fishingGroundCode 漁廠代碼 String(30) Char 30 - V
12 itemColumnConfig 欄位配置 Array - - 定義 R3 的條件式必填欄位 -
13 colName 欄位名稱 String(50) Char 50 欄位識別碼 V
14 colRequired 是否必填 Boolean bit - true=必填，false=選填 V

interface R2Response {

      itemName: string;         // 品名

      itemColumnConfig: Array<{
        colName: string;
        colRequired: boolean;

    }>;

使用方式

Curl 範例

curl -X POST http://192.168.x.x/wmService/v1/PO/PoDetailData \
  -H "Authorization: Bearer {accessToken}" \
  -H "Content-Type: application/json" \
  -d '{
    "requestId": "fb1a6bb8-21cd-490e-9f47-962cf99ec089",
    "wmsAsnNumber": "0000000001",
    "storerCode": "69512619",
    "externReceiptNumber": "API21052600601"
  }'

Android Kotlin 範例

data class ReceivingDetail(
    val lineNo: String,           // WMS 項次 (後續 R3 必用)
    val externLineNo: String,     // PO 項次
    val sku: String,              // 貨號
    val itemName: String,         // 品名
    val expiryDate: String?,      // 到期日
    val fishingGroundCode: String,// 漁廠代碼
    val itemColumnConfig: List<ColumnConfig>
)

data class ColumnConfig(
    val colName: String,
    val colRequired: Boolean
)

class ReceivingRepository {
    private val baseUrl = "http://192.168.x.x/wmService/v1"
    private val client = OkHttpClient()

    suspend fun getReceivingDetails(
        wmsAsnNumber: String,
        storerCode: String,
        externReceiptNumber: String
    ): Result<List<ReceivingDetail>> = withContext(Dispatchers.IO) {
        val requestId = UUID.randomUUID().toString()
        val token = getAccessToken()

        try {
            val jsonRequest = JSONObject().apply {
                put("requestId", requestId)
                put("wmsAsnNumber", wmsAsnNumber)
                put("storerCode", storerCode)
                put("externReceiptNumber", externReceiptNumber)
            }

            val body = jsonRequest.toString().toRequestBody("application/json".toMediaType())
            val request = Request.Builder()
                .url("$baseUrl/PO/PoDetailData")
                .addHeader("Authorization", "Bearer $token")
                .post(body)
                .build()

            val response = client.newCall(request).execute()
            val responseBody = response.body?.string() ?: throw Exception("無回應")

            val jsonResponse = JSONObject(responseBody)

            if (jsonResponse.getString("status") == "S") {
                val detailList = jsonResponse.getJSONObject("data")
                    .getJSONArray("detailList")

                val details = mutableListOf<ReceivingDetail>()
                for (i in 0 until detailList.length()) {
                    val detail = detailList.getJSONObject(i)
                    val columnConfigs = detail.getJSONArray("itemColumnConfig")
                    val configs = mutableListOf<ColumnConfig>()

                    for (j in 0 until columnConfigs.length()) {
                        val config = columnConfigs.getJSONObject(j)
                        configs.add(
                            ColumnConfig(
                                colName = config.getString("colName"),
                                colRequired = config.getBoolean("colRequired")
                            )
                        )
                    }

                    details.add(
                        ReceivingDetail(
                            lineNo = detail.getString("lineNo"),
                            externLineNo = detail.getString("externLineNo"),
                            sku = detail.getString("sku"),
                            itemName = detail.getString("itemName"),
                            expiryDate = detail.optString("expiryDate", null),
                            fishingGroundCode = detail.getString("fishingGroundCode"),
                            itemColumnConfig = configs
                        )
                    )
                }

                Result.success(details)
            } else {
                val message = jsonResponse.getString("message")
                Result.failure(Exception(message))
            }
        } catch (e: Exception) {
            Result.failure(e)
        }
    }

    // 判斷欄位是否必填
    fun isFieldRequired(detail: ReceivingDetail, fieldName: String): Boolean {
        return detail.itemColumnConfig.any {
            it.colName == fieldName && it.colRequired
        }
    }
}

關聯接口

此接口的輸出為 R3 的重要輸入：

下一個接口 關鍵欄位
R3 - 回傳收貨資料 lineNo、externLineNo、itemColumnConfig

欄位配置說明 (itemColumnConfig)

itemColumnConfig 陣列定義了 R3 接口中哪些欄位為必填，哪些為選填：

colName 欄位說明 colRequired 說明
qty 數量 true=必填，false=選填
batchNumber 生產批號 true=必填，false=選填
mfgDate 製造日 true=必填，false=選填
storageStatus 倉別 true=必填，false=選填
stockType 區位 true=必填，false=選填

錯誤代碼

錯誤代碼 HTTP 說明 因應方式
F119 401 Token 驗證失效 重新登入取得 Token
F982 404 採購單不存在 檢查 wmsAsnNumber、externReceiptNumber
F999 500 伺服器異常 聯繫技術支援

注意事項

  1. 關鍵欄位儲存:

    • lineNo: WMS 項次，R3 新增/更新收貨資料時必須傳遞
    • itemColumnConfig: 用於判斷 R3 的哪些欄位為必填

  2. 欄位配置使用:

    • R3 中的 qty、batchNumber 等欄位，需根據對應明細的 itemColumnConfig 判斷是否必填
    • 同一採購單的不同明細項目，可能有不同的必填欄位配置

  3. 使用者介面建議:

    • 以表格或清單形式展示明細
    • 每筆明細顯示：PO 項次、貨號、品名、到期日、漁廠代碼
    • 提供收貨輸入頁面的進入按鈕

  4. 日期格式:

    • expiryDate 格式為 “yyyy/MM/dd”
    • 若為空值，表示該明細項目無預設到期日

  5. 請求順序:

    • 必須先執行 R1 取得採購單清單
    • 用戶選擇採購單後，傳遞 wmsAsnNumber 來呼叫 R2
