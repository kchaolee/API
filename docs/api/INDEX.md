## 關鍵概念

### 通用規範

所有 API 接口遵循以下通用規範：

| 規範 | 說明 |
|------|------|
| **基礎 URL** | `http://192.168.x.x/wmService/v1` |
| **請求格式** | JSON |
| **回應格式** | JSON |
| **認證方式** | Bearer Token (Authorization Header) |
| **Request ID** | 每個請求需傳遞唯一的 GUID |

### 錯誤代碼

| 錯誤代碼 | HTTP | 說明 | 因應方式 |
|---------|------|------|--------|
| F119 | 401 | Token 驗證失效 | 重新登入取得 Token |
| F009 | 500 | JSON 解析失敗 | 檢查請求格式 |
| F981 | 400 | 資料格式錯誤 | 修正請求資料 |
| F982 | 404 | 資料不存在 | 確認資料 |
| F983 | 400 | 業務邏輯錯誤 | 檢查狀態邏輯 |
| F984 | 400 | 資料重複 | 檢查重複請求 |
| F997 | 500 | 伺服器忙碌 | 等待後重試 |
| F999 | 500 | 伺服器處理異常 | 聯繫技術支援 |

### 連貫流程

**進貨作業 (R1 → R2 → R3 → R4 → R5)**:

| 步驟 | 接口 | 說明 | 關鍵輸出欄位 |
|------|------|------|------------|
| 1 | R1 | 取得待收貨採購單清單 | `wmsAsnNumber`, `storerCode`, `externReceiptNumber` |
| 2 | R2 | 取得採購單明細 | `lineNo`, `itemColumnConfig` |
| 3 | R3 | 輸入收貨資料 | - |
| 4 | R4 | 取得核對明細 | `locCode` |
| 5 | R5 | 全收確認過帳 | `palletLabelList` |

## 技術規格文件標準

每個接口技術規格文件包含以下章節：

1. **接口資訊** - URL、方法、格式等基礎資訊
2. **功能說明** - 接口的用途與功能描述
3. **請求參數** - 完整的請求參數表格
4. **回應參數** - 完整的回應參數表格
5. **使用方式** - Curl、Android 範例等實作方式
6. **關聯接口** - 上下游接口關聯
7. **錯誤代碼** - 該接口特有的錯誤代碼說明
8. **注意事項** - 開發時的重點提醒
