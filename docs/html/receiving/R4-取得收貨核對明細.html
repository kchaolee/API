R4 - 取得收貨核對明細

接口資訊

項目 內容
接口名稱 取得收貨核對明細
接口代號 R4
URL POST /wmService/v1/PO/PoVerifying
通訊方式 POST
請求格式 JSON
回應格式 JSON
提供方 WMS
調用方 外部系統 (Android App)

功能說明

完成所有收貨項目後，取得核對資料與儲位選單。顯示每個 PO 項次的收貨完成度、已收件數摘要以及可選擇的儲位清單。

請求參數

項次 參數名稱 參數說明 傳輸型態 DB資料類型 字串最大長度 備註 必填
1 requestId 請求編號 String(36) GUID 36 每個請求唯一，重複提交使用同一編號 V
2 wmsAsnNumber WMS預期收貨單號 String(10) Char 10 帶 R1 回傳的 “WMS預期收貨單號” V
3 storerCode 貨主代碼 String(15) Char 15 帶 R1 回傳的 “貨主代碼” V
4 externReceiptNumber 外部收貨編號 String(30) Char 30 外部收貨編號＝採購單號 V

請求範例

{
  "requestId": "fb1a6bb8-21cd-490e-9f47-962cf99ec089",
  "wmsAsnNumber": "0000000001",
  "storerCode": "69512619",
  "externReceiptNumber": "API21052600601"
}

使用方式

  -H "Authorization: Bearer {accessToken}" \
  -H "Content-Type: application/json" \
  -d '{
    "requestId": "fb1a6bb8-21cd-490e-9f47-962cf99ec089",
    "wmsAsnNumber": "0000000001",

  }'

Android Kotlin 範例

data class VerifyData(
    val warehouseCode: String,
    val wmsAsnNumber: String,
    val storerCode: String,
    val externReceiptNumber: String,
    val poTotalQty: String,      // PO總需求公斤數
    val recTotalPackQty: Int,    // 已收總件數
    val verifyList: List<VerifyDetail>,
    val locList: List<Location>
)

data class VerifyDetail(
    val lineNo: String,         // WMS項次
    val externLineNo: String,   // PO項次
    val sku: String,            // 貨號
    val itemName: String,       // 品名
    val expiryDate: String,     // 到期日
    val recPackQty: Int         // 項次已收件數
)

data class Location(
    val locCode: String         // 儲位編號
)

class ReceivingRepository {
    private val baseUrl = "http://192.168.x.x/wmService/v1"
    private val client = OkHttpClient()

    suspend fun getReceivingVerifyData(
        wmsAsnNumber: String,
        storerCode: String,
        externReceiptNumber: String
    ): Result<VerifyData> = withContext(Dispatchers.IO) {
        val requestId = UUID.randomUUID().toString()
        val token = getAccessToken()

        try {
            val jsonRequest = JSONObject().apply {
                put("requestId", requestId)
                put("wmsAsnNumber", wmsAsnNumber)
                put("storerCode", storerCode)
                put("externReceiptNumber", externReceiptNumber)
            }

            val body = jsonRequest.toString().toRequestBody("application/json".toMediaType())
            val request = Request.Builder()
                .url("$baseUrl/PO/PoVerifying")
                .addHeader("Authorization", "Bearer $token")
                .post(body)
                .build()

            val response = client.newCall(request).execute()
            val responseBody = response.body?.string() ?: throw Exception("無回應")

            val jsonResponse = JSONObject(responseBody)

            if (jsonResponse.getString("status") == "S") {
                val data = jsonResponse.getJSONObject("data")

                // 解析核對明細
                val verifyList = data.getJSONArray("verifyList")
                val details = mutableListOf<VerifyDetail>()
                for (i in 0 until verifyList.length()) {
                    val detail = verifyList.getJSONObject(i)
                    details.add(
                        VerifyDetail(
                            lineNo = detail.getString("lineNo"),
                            externLineNo = detail.getString("externLineNo"),
                            sku = detail.getString("sku"),
                            itemName = detail.getString("itemName"),
                            expiryDate = detail.getString("expiryDate"),
                            recPackQty = detail.getString("recPackQty").toInt()
                        )
                    )
                }

                // 解析儲位選單
                val locList = data.getJSONArray("locList")
                val locations = mutableListOf<Location>()
                for (i in 0 until locList.length()) {
                    val loc = locList.getJSONObject(i)
                    locations.add(
                        Location(
                            locCode = loc.getString("locCode")
                        )
                    )
                }

                val verifyData = VerifyData(
                    warehouseCode = data.getString("warehouseCode"),
                    wmsAsnNumber = data.getString("wmsAsnNumber"),
                    storerCode = data.getString("storerCode"),
                    externReceiptNumber = data.getString("externReceiptNumber"),
                    poTotalQty = data.getString("poTotalQty"),
                    recTotalPackQty = data.getString("recTotalPackQty").toInt(),
                    verifyList = details,
                    locList = locations
                )

                Result.success(verifyData)
            } else {
                val message = jsonResponse.getString("message")
                Result.failure(Exception(message))
            }
        } catch (e: Exception) {
            Result.failure(e)
        }
    }
}

關聯接口

此接口為 R5 之前置接口，輸出的 locCode 為 R5 的必填欄位：

上一個接口 下一個接口
R3 - 回傳收貨資料 R5 - 回傳全收確認

業務邏輯

  1. 資料彙總:

    • poTotalQty: 該採購單的 PO 需求總量（公斤數）
    • recTotalPackQty: 所有項次的已收件數總和

  2. 核對明細:

    • 列出所有 PO 項次的收貨狀況
    • recPackQty: 每個項次已輸入的件數
    • 0 表示該項次尚未輸入收貨資料

  3. 儲位選單:

    • 提供可選擇的儲位清單
    • 用戶從中選擇一個儲位作為 R5 的輸入

錯誤代碼

錯誤代碼 HTTP 說明 因應方式
F119 401 Token 驗證失效 重新登入取得 Token
F982 404 採購單不存在 檢查 wmsAsnNumber、externReceiptNumber
F999 500 伺服器異常 聯繫技術支援

注意事項

  1. 關鍵欄位儲存:

    • locCode: 儲位編號，R5 全收確認時必須傳遞
    • 用戶需要從 locList 中選擇一個儲位

  2. 核對資料展示:

    • 建議以表格形式展示 verifyList
    • 顯示欄位：PO 項次、貨號、品名、到期日、已收件數
    • 高亮顯示未收完的項目（recPackQty = 0）

  3. 進度提示:

    • 顯示已收件數佔總量的百分比
    • 例如：“已收貨 300 件，總需求 500 公斤”

  4. 儲位選擇:

    • 提供下拉選單讓用戶選擇儲位
    • 儲位選擇後，R5 呼叫時傳遞 recLocCode

  5. 請求時機:

    • 建議在所有明細項目都完成收貨輸入後，才調用 R4
    • 避免頻繁調用造成資料不一致

  6. 完整性檢查:

    • R4 只提供核對資料，不會檢查所有項次是否皆有件數
    • R5 會執行完整性檢查，所有 PO 項次必須皆有件數才能過帳
