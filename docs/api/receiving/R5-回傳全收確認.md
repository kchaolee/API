# R5 - 回傳全收確認

## 接口資訊

| 項目 | 內容 |
|------|------|
| **接口名稱** | 回傳全收確認 |
| **接口代號** | R5 |
| **URL** | POST `/wmService/v1/PO/PoCfmReceipt` |
| **通訊方式** | POST |
| **請求格式** | JSON |
| **回應格式** | JSON |
| **提供方** | WMS |
| **調用方** | 外部系統 (Android App) |

## 功能說明

輸入板數與儲位，確認過帳並產生板標籤。此接口會檢查所有 PO 項次是否皆有件數，確認後將採購單狀態變更為「收貨完成」。

## 加密說明

請求與回應內容均需經過 AES 256 加密處理，並使用 `RequestData` 和 `ReturnData` 結構包裹：詳細加密規格請參考: [加密說明](../encryption/加密說明.md)

### Android Kotlin 範例

```kotlin
data class ConfirmReceiptResult(
    val warehouseCode: String,
    val wmsAsnNumber: String,
    val storerCode: String,
    val externReceiptNumber: String,
    val vendorName: String,
    val asnFishingGroundName: String,
    val recPalletQty: Int,
    val asnTotalQty: String,       // 單據總數量
    val asnTotalUom: String,       // 單據總數量單位
    val recTotalPackQty: Int,      // 已收總件數
    val palletLabelList: List<PalletLabel>
)

data class PalletLabel(
    val lblExternReceiptNumber: String,
    val lblPallet: String,
    val lblFishingGroundName: String
)

class ReceivingRepository {
    private val baseUrl = "http://192.168.x.x/wmService/v1"
    private val client = OkHttpClient()

    suspend fun confirmReceipt(
        wmsAsnNumber: String,
        storerCode: String,
        externReceiptNumber: String,
        recPalletQty: Int,
        recLocCode: String
    ): Result<ConfirmReceiptResult> = withContext(Dispatchers.IO) {
        val requestId = UUID.randomUUID().toString()
        val token = getAccessToken()

        try {
            val jsonRequest = JSONObject().apply {
                put("requestId", requestId)
                put("wmsAsnNumber", wmsAsnNumber)
                put("storerCode", storerCode)
                put("externReceiptNumber", externReceiptNumber)
                put("recPalletQty", recPalletQty.toString())
                put("recLocCode", recLocCode)
            }

            val body = jsonRequest.toString().toRequestBody("application/json".toMediaType())
            val request = Request.Builder()
                .url("$baseUrl/PO/PoCfmReceipt")
                .addHeader("Authorization", "Bearer $token")
                .post(body)
                .build()

            val response = client.newCall(request).execute()
            val responseBody = response.body?.string() ?: throw Exception("無回應")

            val jsonResponse = JSONObject(responseBody)

            if (jsonResponse.getString("status") == "S") {
                val data = jsonResponse.getJSONObject("data")

                // 解析板標籤清單
                val labelList = data.getJSONArray("palletLabelList")
                val labels = mutableListOf<PalletLabel>()
                for (i in 0 until labelList.length()) {
                    val label = labelList.getJSONObject(i)
                    labels.add(
                        PalletLabel(
                            lblExternReceiptNumber = label.getString("lblExternReceiptNumber"),
                            lblPallet = label.getString("lblPallet"),
                            lblFishingGroundName = label.getString("lblFishingGroundName")
                        )
                    )
                }

                val result = ConfirmReceiptResult(
                    warehouseCode = data.getString("warehouseCode"),
                    wmsAsnNumber = data.getString("wmsAsnNumber"),
                    storerCode = data.getString("storerCode"),
                    externReceiptNumber = data.getString("externReceiptNumber"),
                    vendorName = data.getString("vendorName"),
                    asnFishingGroundName = data.getString("asnFishingGroundName"),
                    recPalletQty = data.getString("recPalletQty").toInt(),
                    asnTotalQty = data.getString("asnTotalQty"),
                    asnTotalUom = data.getString("asnTotalUom"),
                    recTotalPackQty = data.getString("recTotalPackQty").toInt(),
                    palletLabelList = labels
                )

                Result.success(result)
            } else {
                val message = jsonResponse.getString("message")
                Result.failure(Exception(message))
            }
        } catch (e: Exception) {
            Result.failure(e)
        }
    }
}
```

## 關聯接口

此接口為進貨作業的最後一個接口：

| 上一個接口 | 說明 |
|-----------|------|
| [R4 - 取得收貨核對明細](R4-取得收貨核對明細.md) | 提供 `recLocCode` 作為 R5 輸入 |

## 檢核規則

1. **完整性檢查**:
   - 檢查所有 PO 項次是否皆有件數
   - 若有任何項次件數為 0，回傳 F983 錯誤，訊息包含未有件數的項次代碼

2. **狀態檢查**:
   - 檢查採購單是否已過帳
   - 若已過帳，回傳 F983 錯誤

3. **板數驗證**:
   - `recPalletQty` 必須為整數
   - 若非整數，回傳 F981 錯誤

## 錯誤代碼

| 錯誤代碼 | HTTP | 說明 | 因應方式 |
|---------|------|------|--------|
| F119 | 401 | Token 驗證失效 | 重新登入取得 Token |
| F981 | 400 | 資料格式錯誤 | 檢查 `recPalletQty` 是否為整數 |
| F983 | 400 | 業務邏輯錯誤 | 1. 檢查所有項次是否皆有件數<br>2. 檢查採購單是否已過帳 |
| F999 | 500 | 伺服器異常 | 聯繫技術支援 |

**狀態欄位說明**: S:成功, L:請重新登入, F:失敗

## 注意事項

1. **板標籤處理**:
   - 成功回應會包含 `palletLabelList`，用戶應列印標籤
   - 標籤內容包含採購單號、供應商、板數、儲位、貨號、品名
   - 可多個板合併列印一張標籤

2. **完整性檢查**:
   - R5 執行前，建議用戶端先自行檢查所有項次是否皆有件數
   - 避免呼叫 R5 後才收到錯誤

3. **板數輸入**:
   - `recPalletQty` 為用戶輸入的板數總量
   - 通常為本次收貨所使用的板數

4. **儲位選擇**:
   - `recLocCode` 必須來自 R4 的 `locList`
   - 用戶從 R4 的儲位選單中選擇

5. **狀態變更**:
   - R5 成功後，採購單狀態變更為「收貨完成」
   - 該採購單後續無法再次呼叫 R5
   - 若需修改，需聯繫後台人員

6. **重複提交防護**:
   - 同一採購單的第二個 R5 呼叫會收到 F983 錯誤
   - 用戶端應顯示「此採購單已完成收貨」訊息

7. **列印功能**:
   - 建議在 R5 成功後自動列印板標籤
   - 或提供「重新列印標籤」功能（需後端支援）

8. **異常處理**:
   - 若 R5 呼叫失敗，用戶端應提供「重新嘗試」選項
   - 但務必檢查是否為可重試錯誤（如網路逾時）
