# A1 - 登入驗證

## 接口資訊

| 項目 | 內容 |
|------|------|
| **接口名稱** | 登入驗證 |
| **接口代號** | A1 |
| **URL** | POST `/wmService/v1/Login` |
| **通訊方式** | POST |
| **請求格式** | JSON |
| **回應格式** | JSON |
| **提供方** | WMS |
| **調用方** | 外部系統 (Android App) |

## 功能說明

進行帳號密碼驗證，成功後回傳授權 Token 與過期時間，用於後續 API 請求的認證。

## 請求參數

| 項次 | 參數名稱 | 參數說明 | 傳輸型態 | DB資料類型 | 字串最大長度 | 備註 | 必填 |
|------|---------|--------|--------|-----------|-----------|------|------|
| 1 | requestId | 請求編號 | String(36) | GUID | 36 | 每個請求唯一，重複提交使用同一編號 | V |
| 2 | account | 帳號 | String(50) | VarChar | 50 | - | V |
| 3 | password | 密碼 | String(50) | VarChar | 50 | 建議加密傳輸 | V |

### 請求範例

```json
{
  "requestId": "fb1a6bb8-21cd-490e-9f47-962cf99ec089",
  "account": "user001",
  "password": "password123"
}
```

## 回應參數

| 項次 | 參數名稱 | 參數說明 | 傳輸型態 | DB資料類型 | 字串最大長度 | 備註 | 必填 |
|------|---------|--------|--------|-----------|-----------|------|------|
| 1 | requestId | 請求編號 | String(36) | Char | 36 | 傳回原始請求編號 | V |
| 2 | status | API回應結果代碼 | String(1) | Char | - | 成功代碼：S、失敗代碼：F | V |
| 3 | message | API回應結果訊息 | String(255) | Char | 255 | - | V |
| 4 | data | 回覆資料內容 | Object | - | - | 成功時有值 | - |
| 5 | accessToken | 授權記憶體 | String(128) | VarChar | 128 | 用於後續 API 請求的認證 Token | V |
| 6 | expires | 過期時間 | String(19) | DateTime | - | 格式："yyyy/MM/dd HH:mm:ss" | V |

### 回應範例 (成功)

```json
{
  "requestId": "fb1a6bb8-21cd-490e-9f47-962cf99ec089",
  "status": "S",
  "message": "登入成功",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "expires": "2024-12-11 23:59:59"
  },
  "errors": null
}
```

### 回應範例 (失敗 - 帳號密碼錯誤)

```json
{
  "requestId": "fb1a6bb8-21cd-490e-9f47-962cf99ec089",
  "status": "F",
  "message": "登入失敗",
  "data": null,
  "errors": [
    {
      "code": "F982",
      "message": "帳號或密碼錯誤"
    }
  ]
}
```

### 回應範例 (失敗 - 驗證失效)

```json
{
  "requestId": "fb1a6bb8-21cd-490e-9f47-962cf99ec089",
  "status": "F",
  "message": "失敗-驗證不合法",
  "data": null,
  "errors": [
    {
      "code": "F119",
      "message": "帳號或密碼驗證失敗"
    }
  ]
}
```

## 使用方式

後續所有 API 請求需在 HTTP Header 中加入：

```
Authorization: Bearer {accessToken}
```

### Curl 範例

```bash
curl -X POST http://192.168.x.x/wmService/v1/Login \
  -H "Content-Type: application/json" \
  -d '{
    "requestId": "fb1a6bb8-21cd-490e-9f47-962cf99ec089",
    "account": "user001",
    "password": "password123"
  }'
```

### Android Java 範例

```java
import okhttp3.*;
import org.json.*;

public class LoginExample {
    private static final String BASE_URL = "http://192.168.x.x/wmService/v1";
    private static final MediaType JSON = MediaType.get("application/json; charset=utf-8");
    private OkHttpClient client = new OkHttpClient();

    public void login(String account, String password) {
        String requestId = UUID.randomUUID().toString();

        try {
            JSONObject jsonRequest = new JSONObject();
            jsonRequest.put("requestId", requestId);
            jsonRequest.put("account", account);
            jsonRequest.put("password", password);

            RequestBody body = RequestBody.create(jsonRequest.toString(), JSON);
            Request request = new Request.Builder()
                .url(BASE_URL + "/Login")
                .post(body)
                .build();

            Response response = client.newCall(request).execute();
            String responseBody = response.body().string();

            JSONObject jsonResponse = new JSONObject(responseBody);

            if ("S".equals(jsonResponse.getString("status"))) {
                JSONObject data = jsonResponse.getJSONObject("data");
                String accessToken = data.getString("accessToken");
                String expires = data.getString("expires");

                // 儲存 Token 以供後續使用
                saveAccessToken(accessToken, expires);

                System.out.println("登入成功，Token: " + accessToken);
            } else {
                System.out.println("登入失敗: " + jsonResponse.getString("message"));
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    private void saveAccessToken(String token, String expires) {
        // 儲存 Token 到 SharedPreferences 或其他儲存機制
        SharedPreferences prefs = getSharedPreferences("WMSApp", Context.MODE_PRIVATE);
        SharedPreferences.Editor editor = prefs.edit();
        editor.putString("accessToken", token);
        editor.putString("tokenExpires", expires);
        editor.apply();
    }
}
```

## 注意事項

1. **Token 過期處理**
   - Token 有效期建議為 24 小時
   - 用戶端應在 Token 過期前自動重新登入
   - 或當 API 回傳 F119 錯誤時觸發重新登入

2. **安全性考量**
   - 密碼建議使用 HTTPS 加密傳輸
   - Token 應儲存在安全的位置（如 Android 的 EncryptedSharedPreferences）
   - 避免在 Log 中印出 Token

3. **錯誤處理**
   - F119: 帳號密碼錯誤 → 提示使用者重新輸入
   - F999: 伺服器異常 → 聯繫技術支援
