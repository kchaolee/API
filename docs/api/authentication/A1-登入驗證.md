# A1 - 登入驗證

## 接口資訊

| 項目 | 內容 |
|------|------|
| **接口名稱** | 登入驗證 |
| **接口代號** | A1 |
| **URL** | POST `/wmService/v1/Auth/SignInVerification` |
| **通訊方式** | POST |
| **請求格式** | JSON |
| **回應格式** | JSON |
| **提供方** | WMS |
| **調用方** | 外部系統 (Android App) |

## 功能說明

進行帳號密碼驗證，成功後回傳授權 Token 與過期時間，用於後續 API 請求的認證。

## 加密說明

請求與回應內容均需經過 AES 256 加密處理，並使用 `RequestData` 和 `ReturnData` 結構包裹：

- **請求**: 將 JSON 請求體經 AES 256 加密後，放置於 `RequestData` 欄位中
- **回應**: 收到 API 回應後，從 `ReturnData` 欄位中提取加密內容，並進行 AES 256 解密

詳細加密規格請參考: [加密說明](../encryption/加密說明.md)

## 使用方式

### Curl 範例

```bash
# 外層請求
curl -X POST http://192.168.x.x/wmService/v1/Auth/SignInVerification \
  -H "Content-Type: application/json" \
  -d '{
    "RequestData": "AES加密後的數據"
  }'
```

### Android Java 範例

```java
import okhttp3.*;
import org.json.*;

public class LoginExample {
    private static final String BASE_URL = "http://192.168.x.x/wmService/v1";
    private static final MediaType JSON = MediaType.get("application/json; charset=utf-8");
    private OkHttpClient client = new OkHttpClient();

    public void login(String account, String password) {
        String requestId = UUID.randomUUID().toString();

        try {
            // 1. 準備內層請求數據
            JSONObject innerRequest = new JSONObject();
            innerRequest.put("requestId", requestId);
            innerRequest.put("account", account);
            innerRequest.put("password", password);

            // 2. 將內層請求加密
            String encryptedData = AesHelper.encrypt(innerRequest.toString(), aesKey, aesIv);

            // 3. 準備外層請求
            JSONObject outerRequest = new JSONObject();
            outerRequest.put("RequestData", encryptedData);

            RequestBody body = RequestBody.create(outerRequest.toString(), JSON);
            Request request = new Request.Builder()
                .url(BASE_URL + "/Auth/SignInVerification")
                .post(body)
                .build();

            Response response = client.newCall(request).execute();
            String responseBody = response.body().string();

            // 4. 解析外層回應
            JSONObject outerJsonResponse = new JSONObject(responseBody);
            String encryptedResponse = outerJsonResponse.getString("ReturnData");

            // 5. 解密回應
            String decryptedResponse = AesHelper.decrypt(encryptedResponse, aesKey, aesIv);
            JSONObject jsonResponse = new JSONObject(decryptedResponse);

            if ("S".equals(jsonResponse.getString("status"))) {
                JSONObject data = jsonResponse.getJSONObject("data");
                String accessToken = data.getString("accessToken");
                String expires = data.getString("expires");

                // 儲存 Token 以供後續使用
                saveAccessToken(accessToken, expires);

                System.out.println("登入成功，Token: " + accessToken);
            } else {
                System.out.println("登入失敗: " + jsonResponse.getString("message"));
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    private void saveAccessToken(String token, String expires) {
        // 儲存 Token 到 SharedPreferences 或其他儲存機制
        SharedPreferences prefs = getSharedPreferences("WMSApp", Context.MODE_PRIVATE);
        SharedPreferences.Editor editor = prefs.edit();
        editor.putString("accessToken", token);
        editor.putString("tokenExpires", expires);
        editor.apply();
    }
}
```

## 注意事項

1. **Token 過期處理**
   - Token 有效期建議為 24 小時
   - 用戶端應在 Token 過期前自動重新登入
   - 或當 API 回傳 L 或 F119 錯誤時觸發重新登入

2. **安全性考量**
   - 密碼建議使用 HTTPS 加密傳輸
   - Token 應儲存在安全的位置（如 Android 的 EncryptedSharedPreferences）
   - 避免在 Log 中印出 Token

3. **錯誤處理**
   - F119: 帳號密碼錯誤 → 提示使用者重新輸入
   - F999: 伺服器異常 → 聯繫技術支援

4. **請求結構**
   - 重要：所有 API 請求（除登入接口外）必須使用 `RequestData` 構造包裹加密後的數據
   - 加密前的內層 JSON 應先 minimize（移除多餘空白）再進行加密

5. **回應結構**
   - 重要：所有 API 回應必須從 `ReturnData` 構造中提取加密內容並進行解密
   - 解密後才是實際的回應 JSON
